# debate-tool 辩论机制评估报告

> 基于「幕府崩溃与日本当代走向」两阶段辩论（共10轮75条记录）的实证分析  
> 结合 AGM 信念修正理论、Pragma-Dialectics、Socratic Method、AI Multi-agent Debate 研究

---

## 一、当前机制概述

### 辩论流程

```
[run] 初始化
  │
  ├─ R1: 并行发言（round1_task）
  ├─ R1.5: 串行交叉质询（cross_exam，可选）
  ├─ R2: 并行发言（middle_task，若有质询则含"回应质询"）
  ├─ R2.5: 串行质询（可选）
  ├─ ...
  └─ Rn: 并行发言（final_task）
       └─ Judge: 裁判总结

[resume] 续跑
  ├─ 注入观察者消息（--message）
  ├─ 追加 N 轮（--rounds）
  ├─ 可选质询（--cross-exam [N]）
  └─ Judge: 裁判总结
```

### 立场注入方式（当前）

```python
# runner.py L609-610
f"你是「{d['name']}」，风格为「{d['style']}」。第 {rnd} 轮。\n\n"
f"任务：{task_desc}{constraints_block}"
```

**关键特征**：`style` 字符串全程固定，每轮以同样形式注入 system prompt，**没有信念层次结构，没有"本轮可以修正什么"的显式提示**。

## 二、机制痛点实证分析

### 2.1 立场无法结构化 refine

**代码层面**：
- `stance.py` 只生成 `{name, model, style}` 三字段，无 `nonnegotiables`、`claims`、`belief_layers` 等
- `runner.py` 中 style 是纯文本字符串，每轮原样注入，无轮次间演化追踪
- `core.py` 无任何 claims/belief 数据结构

**辩论实证**：
- GPT-5.2（结构必然派）R3 将"体制崩溃"降级为"政策急转"——这是实质性立场让步，但系统无法捕获、无法反馈给其他辩手
- Kimi（合法性审判官）R3 修正："日本更可能走向…慢燃，但这一状态本身构成合法性侵蚀的特殊形态"——立场演化发生了，但没有结构化记录
- 观察者注入 message 后（第二阶段 R6），换模型的辩手确实改善了论据，但这依赖人工干预而非系统自动检测

**理论对标**（AGM 框架）：
- 当前系统相当于只有 **Expansion**（每轮加入新论据），缺少显式的 **Revision**（修正核心信念）和 **Contraction**（主动放弃某些论点）
- 没有区分 Hard Core（不可让渡）/ Kernel（辅助信念）/ Periphery（外围信念）

### 2.2 越辩越明受阻：伪交锋与结构性停滞

**辩论实证**：
- **伪交锋证据**：多轮中"国际体系现实主义者"与"风险红队预警员"围绕"核威慑是否改变外压性质"反复交锋，但双方均承认某种外部压力存在，分歧仅在"压力是否能被国内政治过滤"——核心分歧从未被明确锁定
- **停滞模式**：R7、R8 出现结构派与红队的第三轮同一论题交锋（"法币工具 vs 石高财政"），内容重合度极高，无新信息注入
- **真正推进的交锋**：质询环节（cross-exam）带来了最有价值的论证突破——如合法性审判官质询现实主义者"你把当代自民党类比为萨长，但萨长是颠覆者，自民党才是幕府——矛盾！"，这是真正的论点突破

**理论对标**（Pragma-Dialectics 四阶段）：
- 当前辩论停留在 **Argumentation 阶段**，缺少 **Opening 阶段**（明确双方起点和分歧所在）和 **Conclusion 阶段**（强制要求收敛判定）
- 停滞信号（话题转移、重复论点）未被自动检测

**AI 研究对标**（Du et al., ICML 2024）：
- 发现辩论深度增加时性能先升后降，存在最优深度——当前5轮×4辩手可能已超过最优深度
- **说服力偏差（CW-POR）**：更会措辞的模型（如 Gemini）可能让正确性较低的论点获胜

### 2.3 各抒己见模式缺失

**当前支持度**：
- `constraints` 字段可注入"承认对方合理观点"类要求——**这是唯一现成 hook**
- `round1_task / middle_task / final_task` 可完全重写为协作模式
- `guide_prompt`（resume 时）可注入协作型指令

**缺失的机制**：
- 无"协作轮"（Collaborative Round）：要求辩手先总结对方最有价值的论点，再补充
- 无"反思暂停"：每N轮后显式要求声明"我可能被说服的条件"
- 无"立场更新声明"：无法强制辩手在认为某论点被推翻时声明 "I update my belief on X"

### 2.4 裁判机制局限

**实证**：
- 第二阶段裁判（kimi-k2.5）因安全过滤直接返回"无法回答该问题"——**单裁判的安全过滤是单点故障**
- 第一阶段裁判总结12000 token，信息密度高但人工提取成本高
- 裁判 prompt 未要求显式列出"各方核心不可让渡信念的对立清单"

**缺失结构**：
```yaml
# 裁判输出中应包含但当前没有的字段
core_disagreements:
  - claim_a: "结构必然派：财政现金流恒等式决定崩溃"
    claim_b: "风险红队：法币主权使现金流约束非硬性"
    verdict: "红队在当前时间段更有支撑，但2030年后可能反转"
    confidence: 0.65
```

## 三、"越辩越明"机制的理论框架

### 3.1 辩论的两种基本模式

| 模式 | 目标 | 适用场景 | 进展判定 |
|------|------|----------|----------|
| **对抗型（Adversarial）** | 击败对方、找出漏洞 | 立场明确对立的议题 | 一方论点被彻底反驳 |
| **协作型（Socratic）** | 共同找到真理、互相完善学说 | 预测型、探索型议题 | 双方共同发现新的分歧或共识 |

当前 debate-tool **只支持对抗型**，而"幕府/日本"辩题是**混合型**（部分争议有对错可辨，部分是纯预测不确定性）。

### 3.2 预测型辩论的特殊需求

预测型辩论的根本在于：
1. **信念（Belief）层面**：各方对世界运行机制有不同的基本假设（如"民主慢变量能否过滤外部压力"），这些是**不可通过单次辩论解决**的 Epistemic disagreement
2. **证据（Evidence）层面**：各方引用的历史事实和当代数据虽然可以核实，但**诠释框架**的差异才是真正的分歧来源
3. **预测（Prediction）层面**：未来不确定，好的辩论应当**锁定分歧的可证伪条件**（若X发生，则A的预测正确）

**当前机制缺陷**：立场（style）是方向性提示，不是结构化信念集——辩手会围绕 style 自由发挥，但**根本性信念分歧从未被显式提炼**。

### 3.3 "越辩越明"的操作化定义

辩论在以下情况下"越辩越明"：
- ✅ 发现了之前未意识到的**核心分歧**（如"动员效率 vs 合法性叙事哪个是第一因"）
- ✅ 某方**修正了一个具体论点**（而非整体立场）
- ✅ 双方**共同确认了一个可证伪指标**（如"2027年防卫增税法案通过"）
- ❌ 双方重复相同论点，只换了措辞
- ❌ 一方被说服但原因不清楚（说服力偏差）
- ❌ 表面共识但核心分歧未处理

## 四、改进方向

### 4.1 立场结构化（核心改进）

**目标**：将 style 从自由文本升级为结构化信念集

```yaml
# 建议的立场结构（不破坏现有 style 文本格式，在 style 基础上扩展）
debaters:
  - name: "结构必然派"
    model: "gpt-5.2"
    style: "强调江户体制石高财政结构锁死..."  # 保留现有
    # 新增可选字段：
    nonnegotiables:
      - "财政现金流恒等式是物理约束，不被货币政策完全消解"
    belief_layers:
      hard_core: "结构矛盾最终必然触发相变"
      auxiliary: "日元贬值会将隐性成本显性化"
      peripheral: "急转时间点在2027年前后"
    falsification_conditions:
      - "若2030年社保和防卫费在无增税情况下均达标，则本立场证伪"
```

**实现路径**：
- 字段完全可选，向后兼容
- 注入 system prompt 时显式列出 nonnegotiables
- 裁判要求对 nonnegotiables 的对立进行专项裁定

### 4.2 辩论模式扩展

**新增 `mode` 字段**：

| 模式 | 说明 | 适用 |
|------|------|------|
| `adversarial`（默认） | 当前行为：攻击对方漏洞 | 立场明确对立 |
| `socratic` | 协作发现：先承认，再补充 | 探索型、预测型 |
| `belief_mapping` | 信念图谱：先确认分歧来源，再辩论 | 根本假设不同时 |

`socratic` 模式的 middle_task 模板：
```
本轮任务（协作模式）：
1. 【承认】指出其他辩手中你认为最有价值的1个论点，解释为何你认为它有价值
2. 【补充】在你认同的方向上，补充1个对方没有提到的新论据
3. 【分歧标注】明确一个你与他人根本性信念不同之处，并说明在什么条件下你会修正
```

### 4.3 进展检测与干预

**自动检测**（无需额外 API 调用）：
- 每轮后计算 Jaccard 相似度（已有早停机制）
- 若相似度过高触发"分歧挖掘"提示而非直接早停

**观察者自动注入模板**（基于当前 message 机制）：
```
[自动元评估]
本轮辩论出现以下信号：
- 高相似度（{{similarity}}）：论点可能出现重复
- 以下核心分歧尚未明确锁定：{{unlocked_disagreements}}

请各辩手在下轮中：
1. 明确声明你与[X辩手]在[Y论点]上的根本性信念分歧
2. 给出该分歧的可证伪条件
```

### 4.4 裁判改进

**结构化输出模板**（新增 core_disagreements 段落）：

```
## 核心信念对立图谱
列出各方不可让渡的信念，并标注对立关系：
- A vs B：[信念A]（A方）vs [信念B]（B方）
  - 证伪条件：若[X]发生，则[A/B]错误
  - 当前证据支持：[A/B/无法判断]
```

**多裁判容错**：
- 主裁判 + 备用裁判（不同供应商）
- 主裁判失败时自动降级到备用

---

## 五、总结：辩论类型与机制匹配矩阵

| 辩论类型 | 当前支持 | 所需改进 |
|----------|----------|----------|
| **事实辩论**（对错可查） | ✅ 良好 | 无 |
| **预测型辩论**（不确定性高） | ⚠️ 部分 | 立场结构化 + 可证伪条件 |
| **价值辩论**（世界观差异） | ⚠️ 部分 | belief_mapping 模式 |
| **协作型辩论**（各抒己见） | ❌ 不支持 | socratic 模式 |
| **超长辩论**（>5轮） | ⚠️ 容易停滞 | 进展检测 + 自动干预 |
