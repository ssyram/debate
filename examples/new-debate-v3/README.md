## 已知未解决问题

### 问题一：LLM 标注跨轮一致性无法保证

系统在多处依赖 LLM 对结构化字段（`axis_scores`、`strength_tier`、协议槽位）的自动提取，
并以跨轮一致率 ≥90% 作为隐含前提。然而，temperature > 0 时同一输入在不同 epoch 产生的
输出存在随机偏差；跨模型版本升级时该偏差更无约束。

当前的处理方式是**暴露而非修复**：通过 `EXTRACTION_INSTABILITY` 标记检测到不一致时，
将容忍区间（epsilon）加倍，以宽容的方式继续运行。这可以防止系统崩溃，但不能消除
标注不稳定对评分和终止判定的实质影响。

**长期缺口**：需要引入结构化规则引擎替代纯 LLM 提取，或建立跨版本回归基准集，
对关键提取点做版本升级前的一致性验证。

### 问题二：高权重轴无证据时的终止性死锁

当 `QuestionFrame` 中某个高权重评估轴在现实文献中缺乏可引用证据时，
对应的 `UNCOVERED_HIGH_WEIGHT_AXIS` 类型 `GapSpec` 会持续阻塞系统终止。
系统核心约束（探索充分性）要求该轴必须被覆盖，而"最大持续轮次"上限（可终止性）
则在轮次耗尽后强制退出——两者在逻辑上存在根本冲突，没有理论层面的消解方案。

当 `BUDGET_EXHAUSTED` 触发退出时，当前输出的结论不含任何降级标注，
读者无法区分"完整覆盖后的结论"与"因预算耗尽被迫截断的结论"。

**待补充的输出规范**：`BUDGET_EXHAUSTED` 触发时，系统应在结论段冠以明确警告：
"受限于未覆盖的高权重轴 `[axis_id]`，以下结论仅基于已覆盖维度的证据，
高权重轴的缺失可能对整体结论产生实质影响。"
