# 讨论议题：v3 四象引擎的处理粒度

## 背景

new-debate-v2 建立了四引擎（广度/深度/精度/清晰度）作为命题级故障恢复状态机的完整体系。
每一个命题从 S0(Draft) 进入，经过 10 个状态的非线性转移，最终到达 S6(Verified) 或 S7(Suspended)。

```
S0(Draft) → S1(Clarify) → S2(TestableClaim) → S3(PrecisionCheck)
                ↑                                    ↓         ↓
         (深度挖出新假设，循环)              S4(DepthProbe)  S5(BreadthProbe)
                                                  ↓              ↓
                                            S7(Suspended)  S9(SchemaChallenge)
                                                               ↓
                                                        S6(Verified)
```

v2 的处理单元是**单个命题**。

## v3 草案的目标

输入：一个问题/议题（可以是「某哲学立场是否成立」「某工程决策是否合理」）
输出：多视角辩护完备的答案体系

v3 草案的顺序：广度引擎（立场张力生成多种候选答案）→ 深度引擎（为每个候选答案生成根基辩护）
→ 精度引擎（加固辩护）→ 清晰度引擎（按需澄清）→ 深广引擎（进一步 break down）

v3 的处理单元是**整个问题**。

## 核心张力

这两个处理粒度如何关联？三种可能性的直觉描述：

- **两层分离**：v3 流水线是外层（问题→引擎序列），v2 状态机是内层（每个引擎内部处理命题）
- **替代**：v3 流水线完全替代了 v2 状态机，命题级不再需要独立状态机
- **统一重组**：存在一个更高层的形式化描述，能统一表达两个处理粒度

这个选择没有显然的答案——它取决于对「什么是引擎的处理职责」和「什么是框架的整合目标」的根本判断。
