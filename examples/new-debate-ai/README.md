## 一、为什么要做这件事

这次讨论的起点是一个实践问题：我们有一套多模型辩论框架（debate-tool）在跑，它产出了一些东西，也暴露了一些根本性的问题。用户 Ssyram 在和朋友聊哲学之后意识到，也许需要看到一些更根本的东西来改进或重构整个辩论 AI 的机制。

**总体目标，用最直接的话说：得到更好、更优质、更符合需求、最重要的是更经受得住考验的结论。**「辩论」这个词不完全准确，更准确的说法是「AI 辅助认知」——目标不是辩论本身，而是通过某种结构化的、多视角的过程，使得我们得到的结论比单一模型能给出的更可靠、更深刻、经过了更多维度的检验。

「经受得住考验」是这里最重要的限定词。

### 已知的失败案例（设计这次讨论的直接驱动）

从之前的幕府辩论等实验中，反复出现的失败模式：

- **幽灵靶子**：辩手 A 在 R2 做了关键让步，但 B 在 R3-R5 仍在攻击已被放弃的旧立场
- **各说各话**：多个模型并行发言，实质性交锋很少，大量内容是独立的「立场陈述」
- **立场无根**：生成的抽象立场（「工程务实派」）太平庸，后续讨论的概率分布也是平庸的
- **辩论赛漂移**：系统设计的格式让模型倾向于输出「格式正确的辩论发言」，而不是真实的推理推进

三方的张力是真实的：Linus 追问「能跑吗」，Ssyram 追问「根基假设是什么」，康德追问「这是认识还是幻觉」。这三个追问指向完全不同的方向，不可能被平均化。

### Ssyram 角色的特殊处理

Linus 和康德已经被大模型训练语料充分理解，不需要额外刻画。但 Ssyram 是一个特定的人，需要详尽的 style 描述：

- **知识背景**：概率程序验证、CEGAR 循环经验、Haskell/Rust/F# 函数式编程思维
- **思维习惯**：从具体失败案例逐层上升，每层必须能往下落地；初始抽象质量决定整个精化循环
- **攻击模式**：精准的「前提的前提」攻击，不是散弹枪
- **底线**：不接受「理论上」来回避落地；对车轱辘话有接近生理性的厌恶（来自 paper-reading pipeline Step 7 的 debug 经历）

## 四、两场讨论设计

### Topic 1：「如果从零重新设计 AI 辅助认知系统，它应该长什么样？」

**背景**：写清楚了总体目标（得到更经受考验的结论）、已知问题（幽灵靶子、各说各话、立场无根、辩论赛漂移）和广度/深度/精度三维框架的来源。不预设核心张力，三方从各自立场直接开始。

### Topic 2：「AI 辅助认知系统中，如何设计参与者立场固定机制？」

**背景**：聚焦参与者立场的质量这一具体设计问题。点出「工程务实派产出的输出和任何真实工程师会说的话都不太像，它是一个平均，不是一个人」。不预设 Ssyram 的方向（「从立场生成到找人」），直接让三方开始。

## 六、最终结论

### Topic 1 核心共识（AI 辅助认知系统架构）

#### v0.1 架构宪法（15 条）

1. Transcript 不是状态，发言块不是认知单元
2. Proxy 不是 Concept，代理指标不能倒灌成概念本体
3. 系统的最小单位是 `ClaimInstance`（带维度上下文的可判定命题），不是自然语言句子
4. 自然语言被永久剥夺主状态机真值的写权限
5. 三层证据契约：Tier-1（物理探针，改真值）/ Tier-2（代理束，改义务）/ Tier-3（抽象理念，仅生成探针义务）
6. Claim Normal Form (CNF)：所有命题在入机前必须结构化为 `metric + threshold + workload + guardrails`
7. Scope Drift 的处理规则：弱反例只能触发「降级（Demotion）」，系统绝不自动补出更窄的正断言
8. Schisma 后旧证据的纪元路由：旧维度永远 100%，新维度永远 0%，不共享真值，只共享检索注意力
9. 不可裁判的命题不配拥有真值——无法映射到物理探针的概念转化为「探针开发义务」
10. 指标改变的路由算法：语义错位 → 操作化缺失 → 操作化竞争 → 价值冲突（移交人类）
11. Goodhart 的防线：Tripod（防尺子烂）+ assertion_ceiling（防说过头）+ Perturbation Audit（防共偏差）
12. InferenceAuditor：LLM 降级为数值输出器，结论由系统注册表模板强制合成
13. Risk Acceptance Override：行动越权通过 DAG 污点追踪计算传播范围，不污染主真值
14. Categorical Veto：系统不排测试优先级，直接拦截缺乏 Tier-1 的高危动作，由业务的碰壁顺序驱动测试算力
15. 系统只输出「认识论方差矩阵」，不提供行动建议，将实践理性决断交还带有具体目的的下游规划器

#### 永久存在的分歧（调节参数）

- **Tier-2 代理的合法性阈值**：「探针独裁」vs「代理泛滥」的永恒张力，v0.1 只能说「代理只能改义务，不能改真值」
- **维度规范化的激进程度**：「图谱爆炸」vs「状态污染」的跷跷板，需在真实业务数据中调参

### Topic 2 核心共识（立场固定机制）

#### 设计立场的正确范式

抽象立场（务实派）和真人模仿（Cosplay Linus）**都是错误的**。正确方向是：

> 提取真实位置的结构性约束：它天然看不见什么（盲区）？它手里有什么杠杆（资源）？它最常犯什么错（误差模式）？它在哪类问题上要求极高证明标准（不对称举证）？

立场的完整结构（「受约束的判断节点」）包含四层：
1. 外部注入的只读宪法（硬底线/终极目的）
2. 认知几何（软权重/搜索预算，即首轮先看什么）
3. 偏执入口（默认怀疑对象与问题重写倾向——「先验危险图型」，而非心理创伤）
4. 误差账本与危机综合程序

#### 当前最可执行的实现方案

在不改动 debate-tool 引擎、只改 `topic.md` 的前提下：

- **格式暴政（Format Tyranny）**：唯一能固定立场的手段是剥夺 LLM 的散文生成权，通过强制的输出数据结构扭曲注意力机制
- **8-12项参与者卡片**，包含：核心责任（终极目的）、身份性失败（最不能接受的事）、首轮操作符（默认先做什么）、改判条件（出现什么时必须让步）、不对称举证门槛（在哪类主张上要求极高证明）、可接受的丑陋（哪些妥协是合理的）
- **独立敌对审查节点**：用温度为 0 的 LLM 实例专门找逻辑漏洞，提供机械补偿路径证伪

#### 评测立场质量的指标

- **ABPI（不对称举证责任指标）**：真正的偏执不是到处抬杠，而是在自己核心防区施加极高但逻辑可满足的证据门槛，在非防区痛快放行
- **「框架破裂与二阶修订测试（Frame-Break & Meta-Revision）」**：代理在连续失败后是否能承认并重写自己的认知框架（而非仅在单次对话中被说服认错）

#### 坦白承认的根本性问题

**「补偿路径的真值问题（经验证实危机）」**：纯文本系统可以检查逻辑自洽、类型匹配，但绝对无法证明一个架构方案在物理世界中真的跑得通。系统能判「像不像一个过得去的方案」，不能闭环判「它在物理世界一定不崩溃」。

## 七、系统集成：完整的 AI 辅助认知系统 MVP

两场讨论在第7轮完成了集成。完整系统的六层架构：

```
Layer 0: 结构化输入
   ↓
Layer 1: OSA（观测面审计）——传感器-主张错配拦截，Hard Block 类型越权
   ↓
Layer 2: Claim Graph + 纪元路由（状态机/记账本）
         - ClaimFamily（索引）/ ClaimInstance（带维度可判定实体）
         - Claim Normal Form：metric + threshold + workload + guardrails
         - Schisma：新维度触发分裂，旧证据纪元隔离
   ↓
Layer 3: 受约束判断节点（施压器）
         - 立场 = 后果位置 + 先验危险图型 + 不对称举证 + 宪法只读
         - 格式暴政：强制输出结构化对象，剥夺散文生成权
   ↓
Layer 4: 敌对审查 + 高摩擦破例协议
         - 反事实穷尽：申请破例必须证明不存在任何替代方案
         - 独立敌对节点只需找出一个替代方案即可一票否决
   ↓
Layer 5: 三层证据契约 + Categorical Veto
         - Tier-1（物理探针，改真值）/ Tier-2（代理束，改义务）/ Tier-3（Obligation）
         - 缺乏 Tier-1 的高危动作被直接拦截，业务碰壁顺序驱动测试算力
   ↓
Layer 6: 认识论尸检报告（输出）
         - killed_paths（被 OSA 击毙的双赢）
         - overturned_vetoes（被敌对审查推翻的否决）
         - collapsed_conflicts（维度不可公度的假冲突）
         - 决策树剪枝源码交还人类，系统不提供行动建议
```

## 九、几个值得记录的元观察

**关于「车轱辘话」被消灭的机制**：每轮注入的 message 都带有「不给具体方案的发言视为无效」的硬性声明。这不只是约束，而是把「认知增量」变成了可检验的输出格式要求。裁判评估中，三方发言里几乎没有出现无效轮次。

**关于「跨学科编译」**：最高质量的推进往往发生在两个不同框架被强制对齐的时候——康德的「先验统觉」被逼着编译成「图数据库的写权限控制」；Linus 的「KISS」被追问成「局部简单 vs 全局简单的分歧」。这种跨层逼迫比单层深挖更有成效。

**关于三角色设计的有效性**：这次三角色的核心贡献是真正不同的——Linus 提供了「这在具体边界条件下会怎样失败」的工程压测；Ssyram 提供了「这个结论能往下落到什么程度、能往上追溯到什么假设」的双向检验；康德提供了「你是否在用工程有用性偷换认识论真理性」的根本性追问。任何两方组合都会失去这种张力。

**关于「经受得住考验」的检验**：10 轮讨论结束后，这套架构被三方从工程边界（Linus）、形式化抽象（Ssyram）、认识论合法性（康德）三个方向都压测过，而且都找到了真实的失效边界。它不是「理论上正确」，而是「已知它在哪里会断」——这已经比大多数架构设计更经受得住考验。
