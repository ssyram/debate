# 元辩论议题二：AI辩论中的立场演化——固定style字段够用吗？

## 议题核心

在 debate-tool 中，每个辩手的立场由 `style` 字段完整描述，这个字段在辩论开始时一次性注入 system prompt，全程不变。当辩手在辩论中"让步"或"深化"时，这个变化**没有被系统捕获，也没有被传达给其他辩手**。

这是不是一个需要解决的问题？如果是，怎么解决？

---

## 系统现状（代码层面）

```python
# runner.py 约 L609-610（核心注入位置）
f"你是「{d['name']}」，风格为「{d['style']}」。第 {rnd} 轮。\n\n"
f"任务：{task_desc}{constraints_block}"
```

**关键特征**：
- `style` 字符串全程固定，每轮以同样形式注入，零修改
- 没有信念层次结构（no `nonnegotiables`，no `belief_layers`，no `claims`）
- 没有"本轮你的任务是：修正/巩固你的以下信念"的显式机制
- `stance.py` 只生成 `{name, model, style}` 三字段，无核心信念结构
- `core.py` 无任何 claims/belief 数据结构

**唯一的现成 refine hook**：
- `resume --message`：观察者可在续跑前注入指令，如"请补强你的薄弱点"
- `round1_task / middle_task / final_task`：可以在任务描述中要求"本轮请声明你的不可让渡信念"
- `constraints` 字段：可注入全局约束，如"每轮须声明在何种条件下你会修正立场"

## AGM 信念修正理论对应分析

AGM 框架（Alchourrón, Gärdenfors & Makinson, 1985）将信念操作分为三类：

| 操作 | 定义 | 当前系统支持度 |
|------|------|--------------|
| **Expansion（扩展）** | 无条件加入新论据 | ✅ 每轮 middle_task 隐性支持 |
| **Revision（修正）** | 在一致性约束下替换信念 | ⚠️ 只能靠辩手自发声明，无系统追踪 |
| **Contraction（收缩）** | 主动放弃某些论点 | ❌ 无任何机制支持，没有辩手明确"收缩"了某个论点 |

**信念层次结构对应**：
- **Hard Core（硬核/不可让渡）**：结构派始终坚持的"结构约束必然推动政策相变"；风险红队始终坚持的"现代国家工具箱与幕府本质不同"
- **Kernel（核心辅助信念）**：结构派的"三重锁死"（少子高龄化/债务刚性/增长停滞）；合法性派的"合法性是资源提取的通道"
- **Periphery（外围可让步）**：结构派"体制崩溃"→R2已让步为"政策级断裂"；"金融正常化是触发点"→R5进一步精确化

**系统缺口**：没有任何结构可以区分这三个层次。结构派在 R2 放弃了外围信念（体制崩溃），但系统对这个放弃一无所知，其他辩手继续针对已放弃的论点发起攻击，白白浪费了 R3-R5 的多个质询轮次。

## 设计方案对比

### 方案A：结构化信念层次（nonnegotiables + belief_layers）

```yaml
debaters:
  - name: "结构必然派"
    model: "gpt-5.2"
    style: "强调江户体制石高财政结构锁死..."  # 保留现有
    # 新增可选字段：
    nonnegotiables:
      - "结构约束最终必然推动政策相变，工具箱更大只改变形态不改变规律"
    belief_layers:
      hard_core: "结构矛盾在外压下触发相变"
      auxiliary: "三重锁死（少子/债务/低增长）构成当代版石高失配"
      peripheral: "政权会崩溃（已让步：修正为政策级断裂）"
    falsification_conditions:
      - "若2030年社保和防卫费在无增税情况下均达标，则本立场证伪"
```

**优点**：结构化，其他辩手和裁判可以基于 nonnegotiables 进行针对性攻防
**缺点**：需要用户在辩题文件中手动填写；如果辩手实际上没有按此结构发言，字段形同虚设

### 方案B：观察者注入机制强化（自动 message 生成）

当前 `--message` 需要人工编写。若系统能自动检测以下模式并生成 message：
- 某辩手的论点被 ≥2 次指出相同缺陷但未回应 → 自动注入："[辩手名] 需要正面回应[缺陷描述]"
- 相似度检测触发停滞 → 自动注入"分歧挖掘"提示

**优点**：不需要修改辩题文件格式，向后兼容
**缺点**：自动检测"相同缺陷"需要额外的 LLM 调用（成本），可能误判

### 方案C：middle_task 中增加强制声明

在内置 middle_task 模板中增加：
```
【立场声明】本轮请明确：
- 你的核心不可让渡信念是什么（一句话）
- 上一轮交锋是否让你修正了某个具体论点（若是，请明确说明修正了什么）
- 你的立场在什么条件下会被证伪
```

**优点**：零工程成本，只修改提示词，立即可用
**缺点**：只是要求辩手声明，不能强制；裁判也需要相应调整来追踪这些声明

## 元问题：AI没有真实信念，refine机制有意义吗？

**虚无主义挑战**：AI模型的每轮输出都是基于上下文的文本预测，所谓"让步"只是模型看到被反驳后生成了更符合对话逻辑的文本。这和人类的"信念修正"在认知层面完全不同。

**可能的回应**：即使没有真实信念，结构化的"声明机制"仍然有工程价值：
- 强制辩手每轮明确"我的核心主张是X"，使模型的输出更聚焦
- 让其他辩手能够针对性地攻击明确的命题（而非模糊的方向）
- 让裁判能够追踪"哪些论点在辩论中被真正回应了，哪些被回避了"

**但虚无主义也有一个无法被轻易反驳的核心**：如果模型可以在下一轮"忘记"自己刚才做出的让步（甚至在同一轮内前后矛盾），那么任何声明机制都只是增加了提示词约束，而非真正的信念修正。
