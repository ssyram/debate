## [A3] 助手回复（续跑策略 + 完整命令）

会，而且 `--message` 很关键。我会优先 `resume --message`，因为它会：

- 先被写入观察者日志；
- 在续跑第一轮触发“请回应观察者意见”；
- 进入裁判总结的“观察者意见回应”部分。

即：它能同时影响**辩手与裁判**。但我会让“倾斜”偏向**未解决争点的判别标准**，而不是偏向某一派胜负。

### 我给的三组命令（完整）

#### 题 1：打实归因（message / 结构约束 / 模型更换）

```bash
cd /Users/ssyram/workspace/github/debate

python3 -m debate_tool resume examples/v2-refine-meta-debate/meta-debate-topic1.md \
  --rounds 2 \
  --cross-exam -1 \
  --message "【主持人续跑要求】本轮禁止继续泛谈“AI能不能越辩越明”。请所有辩手只围绕一个问题：R5→R6质量跃升的主因究竟是什么——(A) message 的最新上下文注入，(B) 交叉质询/结构化约束，还是 (C) 模型更换。要求：1）每位辩手必须明确选择主因与次因；2）必须引用幕府案例中的至少两个具体切片（如R2→R5错靶、R5→R6观察者注入、R7.5或R9.5高质量质询）；3）必须给出一个最小对照实验设计，说明如何区分 A/B/C；4）若你认为多个因素共同作用，也必须说明哪个是必要条件、哪个只是放大器。裁判请不要按措辞完整性泛判，请优先裁定：目前证据最能支持哪一种归因，哪一种仍属未被证实。"
```

#### 题 2：把 A/B/C 压成最小可比实现

```bash
cd /Users/ssyram/workspace/github/debate

python3 -m debate_tool resume examples/v2-refine-meta-debate/meta-debate-topic2.md \
  --rounds 2 \
  --cross-exam 2 \
  --message "【主持人续跑要求】停止重复争论“AI有没有真实信念”这一元问题；本轮只比较三种实现路线的工程性价比：A=belief_layers/round_concessions，B=Meta-Agent 动态注入，C=middle_task 强制声明 + 轻量公告板。所有辩手必须围绕三个 failure cases 回答：1）幽灵攻击（打旧靶），2）死不开口（长期不交代可证伪条件），3）假让步/换壳逃逸。要求：每位辩手必须提交一个‘最小可实现方案’，精确到字段名、hook 或判罚逻辑；同时指出自己方案最容易被利用的一个漏洞。裁判请按“最小额外复杂度下改善最大”进行裁定，不要只按理论完整性打分；若没有任何方案能同时覆盖三类 failure case，请明确裁定‘当前仍无足够好的单一方案’。"
```

#### 题 3：先降噪（modify）再续跑（resume）


补充结论：三题里若要“同时影响辩手和裁判”，优先用 `--message`；`--guide` 更偏辩手侧软引导。
